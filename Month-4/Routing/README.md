# Домашнее задание: Vagrant-стенд c сетевой лабораторией

## Цель
Научиться менять базовые сетевые настройки в Linux-based системах. Настроить статическую маршрутизацию и NAT.

---

## Текст задания

1. Развернуть Vagrant-стенд с 7 виртуальными машинами.
2. Построить сетевую архитектуру:

**Сеть office1:**
- 192.168.2.0/26 — dev
- 192.168.2.64/26 — test servers
- 192.168.2.128/26 — managers
- 192.168.2.192/26 — office hardware

**Сеть office2:**
- 192.168.1.0/25 — dev
- 192.168.1.128/26 — test servers
- 192.168.1.192/26 — office hardware

**Сеть central:**
- 192.168.0.0/28 — directors
- 192.168.0.32/28 — office hardware
- 192.168.0.64/26 — wifi

## Особенности реализации

- **NAT**: настроен на `inetRouter` через iptables MASQUERADE. Правила сохраняются через `/etc/network/if-pre-up.d/iptables` и применяются при старте сети.
- **IP Forwarding**: включён на всех 4 роутерах через `sysctl net.ipv4.conf.all.forwarding=1`.
- **Отключение default route на eth0**: на всех хостах кроме `inetRouter` в `/etc/netplan/00-installer-config.yaml` добавлено `dhcp4-overrides: use-routes: false`.
- **Статические маршруты**: прописаны в `/etc/netplan/50-vagrant.yaml` для каждого хоста. Шаблоны: `50-vagrant_<hostname>.yaml`.
- **Обратные маршруты**: `centralRouter` знает пути к сетям office1 (`192.168.2.0/24`) и office2 (`192.168.1.0/24`). `inetRouter` знает путь к `192.168.0.0/16` для возврата NAT-трафика.
- **Запуск Ansible**: происходит после создания последней VM (`office2Server`) командой `vagrant up`.

## Проверка

```bash
# Трассировка интернета с office1Server
vagrant ssh office1Server
traceroute 8.8.8.8
# Должно идти: office1Router → centralRouter → inetRouter → Internet

# Ping между офисами
ping 192.168.1.2   # office2Server из office1Server
ping 192.168.0.2   # centralServer из office2Server
```
